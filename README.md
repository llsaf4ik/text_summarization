# Сравнение архитектур Transformer и LSTM в задаче суммаризации текстов

## О проекте

Данный проект представляет собой исследовательскую работу в области обработки естественного языка (NLP). Главная цель проекта – глубокое изучение, низкоуровневая реализация и сравнение двух фундаментальных архитектур глубокого обучения (Sequence-to-Sequence): **Transformer** и **Bidirectional LSTM + Attention**. 

Теоретическое описание и разбор архитектур доступны в конспектах: **[Transformer](https://github.com/llsaf4ik/ml_notes/blob/main/Transformer.ipynb)** и **[LSTM](https://github.com/llsaf4ik/ml_notes/blob/main/RNN.ipynb)**.  Здесь же представлена практическая реализация исследования.

## Ключевые инженерные особенности реализации

В рамках проекта не использовались готовые реализации архитектур, такие как `nn.Transformer` или `nn.MultiheadAttention`, все архитектуры были созданы с нуля на базе фреймворка PyTorch. Это позволило внедрить ряд структурных оптимизаций:

1. **KV-Cache:** В архитектуру декодера Трансформера интегрирован механизм кэширования тензоров $K$ и $V$, который сильно ускоряет процесс инференса, так как вычислительная сложность на каждом шаге генерации текста снижается с $O(n^2)$ до $O(n)$.

2. **Динамическое маскирование и оптимизация RNN:** Для модели LSTM применена техника упаковки тензоров переменной длины (`pack_padded_sequence` и `pad_packed_sequence`). Это предотвращает выполнение лишних вычислительных операций на padding-токенах. Также реализован механизм внимания с маскированием padding-токенов, который исключает влияние отступов на итоговый результат.

3. **Стратегии обучения:** Обе архитектуры обучались с применением Teacher Forcing, однако для LSTM дополнительно был интегрирован алгоритм **Scheduled Sampling**: на каждом шаге декодер с заданной вероятностью получает на вход либо истинный токен из обучающей выборки, либо собственный предсказанный токен с предыдущего шага. Это сделано для того, чтобы минимизировать расхождение распределения данных во время обучения и инференса.

4. **Штрафование за повторения:** Для улучшения качества генерации текста Трансформера была применена техника штрафования модели за повторы. Параметр `repetition_penalty` снижает вероятность сгенерировать тот же токен, если он уже был сгенерирован ранее, а параметр `penalty_window` задает окно штрафа, внутри которого действует это правило. Такой подход заставляет модель выбирать более разнообразные слова и позволяет гораздо реже попадаться в ситуации, когда генерируется один и тот же токен.

5. **Аугментации:** Для повышения робастности моделей и предотвращения переобучения были реализованы классы текстовых аугментаций: случайная замена слова его синонимом и случайная перестановка слов, стоящих рядом.

6. **Pre-LN vs Post-LN:** Изначально архитектура Трансформера была выстроена в соответствии с **Post-LN**, что приводило к низким метрикам качества. Изменение архитектуры на **Pre-LN** позволило стабилизировать обучение модели. Только одно это изменение улучшило финальное качество модели более чем в 2 раза.

## 📊 Данные и стратегия обучения

В качестве основного корпуса текстов для обучения моделей использовался русскоязычный датасет **Gazeta**, специфика которого заключается в значительной длине исходных новостных текстов и высокой степени абстрактивности аннотаций.

**Стратегия обучения:**
В связи с тем, что архитектура Transformer обучается с нуля и требует большого объема данных, применялось двухэтапное обучение:

1. **Pre-training:** Предварительное обучение на достаточно большом корпусе текстов для суммаризации (~ 500 тыс пар) со средним качеством разметки. На этом этапе модель выучила базовое понимание языка.

2. **Fine-tuning:** Дообучение на высококачественном датасете **Gazeta** (~ 67 тыс пар) с использованием оптимизатора AdamW и стратегии изменения learning rate OneCycleLR. Для предотвращения переобучения на малом объеме данных применялся обширный набор регуляризации: сглаживание меток (`label_smoothing`), затухание весов (`weight_decay`), `dropout` с высокой вероятностью зануления и текстовые аугментации.

## Результаты и оценка качества

Качество моделей оценивалось с использованием стандартного набора метрик **ROUGE**, измеряющих степень перекрытия n-грамм между предсказанием модели и эталонным текстом.

| Модель | ROUGE-1 (Униграммы) | ROUGE-2 (Биграммы) | ROUGE-L (Структура) |
| :--- | :---: | :---: | :---: |
| **LSTM + Attention** | 0.0452 | 0.0146 | 0.0447 |
| **Transformer** | **0.2052** | **0.0624** | **0.2009** |

### Аналитический вывод

Полученные метрики наглядно демонстрируют фундаментальное ограничение рекуррентных нейронных сетей (RNN) при работе с длинными последовательностями:

- Архитектура **LSTM** подвержена проблеме "бутылочного горлышка": вектор скрытого состояния фиксированной размерности не способен эффективно аккумулировать контекст всего новостного текста (до нескольких тысяч токенов), что приводит к эффекту забывания и критическому падению качества суммаризации. Механизм внимания лишь частично нивелирует эту проблему.

- В свою очередь, архитектура **Transformer**, благодаря механизму Self-Attention, обладает глобальным рецептивным полем. Модель вычисляет попарное сходство между всеми токенами одновременно, получая прямой доступ к любой части исходного текста независимо от дистанции между словами. Это позволило ей превзойти рекуррентную архитектуру в 4 раза по целевой метрике.

## Запуск инференса

Вы можете протестировать работу Трансформера локально с помощью подготовленного консольного скрипта.

1. **Склонируйте репозиторий:**
    ```bash
    git clone git@github.com:llsaf4ik/text_summarization.git
    cd text_summarization
    ```
2. **Создайте и активируйте виртуальное окружение:**
    ```bash
    python -m venv venv
    ```
    для `linux/macos`:
    ```bash
    . venv/bin/activate
    ```
    для `windows`:
    ```bash
    . venv/Scripts/activate
    ```
3. **Установите необходимые зависимости:**
    ```bash
    pip install -r requirements.txt
    ```
4. **Скачайте веса модели:**

	Ссылка на [Releases](https://github.com/llsaf4ik/text_summarization/releases/download/v1.0/transformer.pth). Поместите веса модели рядом с файлом `predict.py`.
	
5. **Запуск генерации:**

	Используйте готовый вариант или создайте свой файл `input.txt` и вставьте в него текст для суммаризации. Файл должен быть в той же директории, где расположен `predict.py`.
	```bash
    python predict.py --input input.txt --output summary.txt
    ```
    Сгенерированная аннотация будет сохранена в файл `summary.txt`.
    
	**Дополнительные параметры скрипта:**

	-   `--input` или `-i`: Путь к исходному файлу (обязательный аргумент).
    
	-   `--output` или `-o`: Имя выходного файла (по умолчанию: `summary.txt`).
    
	-   `--weights` или `-w`: Путь к файлу весов `.pth` (если он находится в другой папке).
    
	-   `--tokenizer` или `-t`: Путь к JSON-файлу токенизатора.
